## Issues Fixed

### 1. HNSW Search Bug (src/similarity/index.py)

**Problem:** Single-item searches returned `-inf` similarity because the entry point distance was initialized to `float('inf')` instead of computing the actual distance.

**Fix:**
```python
# Before (line 183-187)
if self.entry_point is None:
    return []

nearest = [(float('inf'), self.entry_point)]

# After
if self.entry_point is None:
    return []

entry_embedding = self.data[self.node_reverse_map[self.entry_point]]
entry_distance = self._compute_distance(query, entry_embedding)
nearest = [(entry_distance, self.entry_point)]
```

### 2. IndexConfig Parameter Mismatch (src/api/main.py)

**Problem:** `IndexConfig` was called with invalid parameters (`max_elements`, `ef_construction`, `ef_search`, `space`).

**Fix:**
```python
# Before
index_config = IndexConfig(
    dimension=384,
    max_elements=100000,
    m=16,
    ef_construction=200,
    ef_search=100,
    space="cosine"
)

# After
index_config = IndexConfig(
    dimension=384,
    m=16,
    ef=200,
    metric=SimilarityMetric.COSINE
)
```

### 3. Double Tenant-ID Prefix Bug (src/cache/cache_manager.py)

**Problem:** `put_semantic()` created keys like `tenant-1:tenant-1:hash` because it passed `cache_key` (already prefixed) to `index_manager.add()` which added the prefix again.

**Fix:**
```python
# Before (line 741)
self._index_manager.add(
    item_id=cache_key,  # Already has tenant prefix
    ...
    tenant_id=tenant_id,  # Index manager adds prefix again
)

# After
self._index_manager.add(
    item_id=query_hash,  # No prefix, let index manager add it
    ...
    tenant_id=tenant_id,
)
```

### 4. Test Fixture Issues (tests/integration/test_semantic_integration.py)

**Problems:**
- `IndexConfig` fixture used wrong parameters
- `get_semantic()` calls missing required `query_text` parameter
- Search parameter named `query_embedding` instead of `embedding`
- Stats assertions used wrong keys
- Async fixture handling issues

**Fixes:**
- Updated `index_config` fixture with correct parameters
- Added `query_text` to all `get_semantic()` calls
- Changed `query_embedding=` to `embedding=`
- Updated stats assertions to match actual keys (`semantic_hits`, `semantic_misses`, etc.)
- Made `embedding_service` fixture synchronous with graceful skip on failure

### 5. Config Test Environment Variable Override (tests/unit/test_phase1_1_foundation.py)

**Problem:** Test expected YAML config value but `.env` file had `API_HOST=0.0.0.0` which overrode it.

**Fix:**
```python
# Added monkeypatch to clear env vars
def test_config_loader_from_yaml(self, monkeypatch):
    monkeypatch.delenv("API_HOST", raising=False)
    monkeypatch.delenv("API_PORT", raising=False)
    # ... rest of test
```
